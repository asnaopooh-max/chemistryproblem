# questions.json スキーマ

このファイルは `questions.json` の形式を説明します。アプリはこの JSON を読み込み、選択問題を出題します。

## フィールド

- `id` (文字列、必須)
  - 問題を一意に識別する ID。
  - 例: `CB-001`

- `category` (文字列、必須)
  - `constants.js` に定義された分野名と一致させる。
  - 例: `物質の構成`

- `unit` (文字列、任意)
  - 問題の単元や単元名。
  - 例: `物質量`

- `topic` (文字列、任意)
  - 問題のテーマ名。
  - 例: `molと粒子数`

- `difficulty` (数値、必須)
  - 1〜5 の難易度。
  - 1: 基礎確認、2: 標準、3: 共通テスト基礎、4: 共通テスト標準、5: 共通テスト発展。

- `skill` (文字列、任意)
  - 計算、理解、読解、判断など。

- `question` (文字列、必須)
  - 問題文。簡単な HTML タグを含めることもできる。

- `choices` (配列、必須)
  - 選択肢の配列。5〜8 個であること。

- `answerIndex` (数値、必須)
  - 正解の選択肢の添字。0 始まり。

- `explanation` (文字列、必須)
  - 正答・誤答後に表示される解説。

- `point` (文字列、任意)
  - 解説に加えた学習ポイント。

- `extra` (文字列、任意)
  - 問題文の追加資料や補足データを表示する場合に使用。

## 注意点

- `choices` は必ず 5〜8 個にする。
- `answerIndex` は `choices` の範囲内である必要がある。
- `category` は `constants.js` の分野一覧と一致させる。
- 数式はまずはテキスト表記で記述し、後から MathJax などに対応しやすい設計にする。
- 保存やサーバー通信を行わないので、問題の状態はすべて JavaScript 実行中のメモリ上で管理される。
